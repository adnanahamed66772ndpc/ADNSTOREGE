# R2 বাকেট ও কী কিভাবে সেটআপ করবেন

অ্যাপ ফাইল জমা রাখে Cloudflare R2 বাকেটে। প্রতিটি বাকেট ও তার কী অ্যাপের **Settings** থেকে যোগ করবেন (env ভেরিয়েবল লাগে না)। এই গাইডে R2 বাকেট তৈরি এবং **endpoint**, **access key**, **secret key** কোথায় পাবেন সেটা বলা আছে।

---

## ১. R2 বাকেট তৈরি করা

1. [Cloudflare Dashboard](https://dash.cloudflare.com) এ গিয়ে লগইন করুন।
2. বাম পাশে **R2** খুলুন ("Storage" বা "Object Storage" এর নিচে)।
3. **Create bucket** ক্লিক করুন।
4. **বাকেটের নাম** দিন (যেমন `my-files`)। মনে রাখুন; অ্যাপে "Bucket name" হিসেবে এটা দেবেন।
5. জায়গা বেছে নিতে বললে নিতে পারেন (ঐচ্ছিক)।
6. **Create bucket** ক্লিক করুন।

এখন একটা বাকেট আছে। পরের ধাপে অ্যাপ যাতে পড়া-লেখা করতে পারে তার জন্য API ক্রেডেনশিয়াল নিতে হবে।

---

## ২. R2 API কী (access key ও secret key) নেওয়া

R2 **API টোকেন** দিয়ে **Access Key ID** আর **Secret Access Key** (S3-সামঞ্জস্যপূর্ণ) পাওয়া যায়।

1. ড্যাশবোর্ডে **R2** এ যান।
2. ডান পাশের মেনুতে **Manage R2 API Tokens** (বা **Overview** → **Manage API Tokens**) ক্লিক করুন।
3. **Create API token** ক্লিক করুন।
4. টোকেনের নাম দিন (যেমন `adnstorge-app`)।
5. **Permissions:** বেছে নিন:
   - **Object Read & Write** (যাতে অ্যাপ আপলোড, ডাউনলোড, ডিলিট করতে পারে), অথবা
   - পুরো R2 এক্সেস চাইলে **Edit**।
6. চাইলে শুধু নির্দিষ্ট বাকেটে সীমাবদ্ধ রাখতে পারেন (যেমন শুধু `my-files`)।
7. **Create API token** ক্লিক করুন।
8. পরের পেজে দেখাবে:
   - **Access Key ID** (লম্বা স্ট্রিং)
   - **Secret Access Key** (লম্বা স্ট্রিং)

**মনে রাখুন:** দুটোই কপি করে নিরাপদে রাখুন। **Secret Access Key** শুধু একবার দেখায়। হারিয়ে গেলে নতুন API টোকেন বানাতে হবে।

---

## ৩. R2 endpoint (S3 API endpoint) কোথায় পাবেন

অ্যাপের আপনার অ্যাকাউন্টের **S3-সামঞ্জস্যপূর্ণ API endpoint** দরকার যাতে R2-এর সাথে কথা বলতে পারে।

1. ড্যাশবোর্ডে **R2** এ যান।
2. **Manage R2 API Tokens** ক্লিক করুন (উপরে যেখানে গিয়েছিলেন)।
3. সেই পেজে **S3 API** বা **Endpoint** এর তথ্য থাকবে।
4. endpoint সাধারণত এরকম:
   - `https://<ACCOUNT_ID>.r2.cloudflarestorage.com`
   - অথবা "S3 API endpoint" হিসেবে অ্যাকাউন্ট আইডি সহ দেখায়।

**Account ID কোথায় পাবেন:**

- ড্যাশবোর্ডে R2 বা অন্য কোনো Cloudflare পেজে থাকার সময় URL দেখুন:  
  `https://dash.cloudflare.com/<ACCOUNT_ID>/r2/...`
- অথবা **Workers & Pages** → **Overview** এ যান; সেখানে অ্যাকাউন্ট আইডি থাকে।

তারপর endpoint হবে:

```text
https://<YOUR_ACCOUNT_ID>.r2.cloudflarestorage.com
```

উদাহরণ: অ্যাকাউন্ট আইডি যদি `abc123def456` হয়, endpoint:

```text
https://abc123def456.r2.cloudflarestorage.com
```

## ৪. অ্যাপে বাকেট যোগ করা

1. অ্যাপ খুলে **Settings** এ যান।
2. "Add bucket" (বা "Edit bucket") এ এইগুলো পূরণ করুন:

| ফিল্ড | কী লিখবেন |
|-------|------------|
| **Display name** | যেকোনো লেবেল (যেমন "আমার স্টোরেজ")। |
| **R2 endpoint** | `https://<ACCOUNT_ID>.r2.cloudflarestorage.com` |
| **Access key** | R2 API টোকেনের **Access Key ID**। |
| **Secret key** | R2 API টোকেনের **Secret Access Key**। |
| **Bucket name (R2)** | R2 এ যে নামে বাকেট বানিয়েছেন ঠিক সেটা (যেমন `my-files`)। |
| **Storage limit (GB)** | যেমন `9` (অ্যাপে বাকেট প্রতি লিমিট)। |

3. **Add bucket** (বা এডিট করলে **Update**) ক্লিক করুন।

অ্যাপ S3 API দিয়ে R2-এর সাথে কানেক্ট করবে। আরও বাকেট যোগ করতে পারেন (একাধিক অ্যাকাউন্টে একাধিক API টোকেনও ব্যবহার করতে পারেন)।

---

## ৫. দ্রুত রেফারেন্স: কী কোথায় পাবেন

| যা দরকার | কোথায় পাবেন |
|----------|----------------|
| **Bucket name** | R2 → আপনার বাকেট → উপরের নাম (যেমন `my-files`)। |
| **Access key** | R2 → Manage R2 API Tokens → Create API token → **Access Key ID** (যেই মুহূর্তে দেখায় কপি করুন)। |
| **Secret key** | একই পেজে **Secret Access Key** (একবারই দেখায়, কপি করে রাখুন)। |
| **Endpoint** | `https://<ACCOUNT_ID>.r2.cloudflarestorage.com`। **Account ID** ড্যাশবোর্ড URL বা R2/Workers ওভারভিউ থেকে নিন। |

---

## ৬. একাধিক বাকেট বা একাধিক Cloudflare অ্যাকাউন্ট

- **একই অ্যাকাউন্টে কয়েকটা বাকেট:** সব বাকেটে এক্সেস আছে এমন একটা API টোকেন বানান (অথবা বাকেট প্রতি একটা)। সবার জন্য **endpoint** একই; শুধু **bucket name** (এবং চাইলে **access key** / **secret key**) বাকেটভেদে বদলাবে।
- **আলাদা Cloudflare অ্যাকাউন্ট:** ওই অ্যাকাউন্টে বাকেট ও API টোকেন বানিয়ে অ্যাপে আরেকটা "বাকেট" যোগ করুন সেই অ্যাকাউন্টের **endpoint**, **access key**, **secret key**, **bucket name** দিয়ে।

এগুলো সব অ্যাপের **Settings** থেকে কনফিগ করা; কোড বা env ভেরিয়েবল লাগে না।
